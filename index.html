What's New in v2.0

Ultra Low Bandwidth
& Major Upgrades

The biggest update since launch. Every change reduces bandwidth, improves security, or adds user-requested features.

Ultra Low Bandwidth New

Firebase listener reduced from `limitToLast(100)` to `limitToLast(5)`. 95% less data transferred. Your own messages are instantly deleted from the relay without waiting for the listener callback.

Bandwidth

Password Toggle New

Eye icon on the password field toggles between hidden and visible. Uses a clean button overlay with `fa-eye` / `fa-eye-slash` icon swap. No alert, no prompt — just a click.

UX

Auto-Cleanup on Disconnect New

`onDisconnect().remove()` on the status node ensures your online presence is wiped the moment you close the tab, lose connection, or navigate away. No ghost online dots.

Reliability

Per-Chat Import New

Import JSON chats directly into a specific conversation via the contact context menu ("⋯" → Import). Only messages matching that chat's UID pair are inserted. No more global import clutter.

Backup

60-Second Status Heartbeat New

Status `lastSeen` updates every 60 seconds instead of on every action. Massive reduction in Firebase write operations. Online status is set once on login, then heartbeated.

Performance

XSS-Safe Escaping New

All user-generated text (usernames, messages, filenames) passes through `esc()` which uses `textContent` assignment then reads `innerHTML` — a DOM-based sanitizer that neutralizes injection attacks.

Security

UUID Deduplication Guard New

Before rendering any bubble, `addBubble()` checks `document.getElementById('msg-'+m.id)`. If the message already exists in the DOM, it's silently skipped. Prevents duplicate bubbles from race conditions.

Reliability

Proper Cleanup Function New

New `cleanup()` function handles: clear status interval, detach message listener, set offline, remove status node. Called on both logout and account deletion — no orphaned listeners or stale data.

Stability

3MB File Limit Changed

File size limit reduced from 5MB to 3MB to stay within Firebase's optimal realtime payload range. Keeps relay purge fast and IndexedDB responsive even on mobile.

Security Architecture

8 Layers of
Protection

Every design decision starts with: "Does this increase attack surface?" v2.0 adds two new layers.

Level 5

AES-256-GCM Key Generation

`crypto.subtle.generateKey()` creates a 256-bit key on registration. Exported as base64, stored in user profile for future E2E layer.

Level 5

Device-Only Storage (hgz7)

IndexedDB version hgz7. Object stores: `m` (messages, keyPath: id) and `k` (keys, keyPath: id). Zero server-side message persistence.

Level 4

Firebase Auth (Compat v10.12.2)

Email/password auth with JWT auto-refresh. Username→UID mapping via `/unames/` node. Password reset via Gmail.

Level 4

Ultra-Low Relay (limitToLast 5)

Only 5 Firebase slots monitored. 95% less data than v1. Self-messages purged without IDB check. Incoming messages deduplicated by UUID.

span class="sec-level l3">Level 3

XSS-Safe DOM Escaping

`esc()` function: creates a div, sets `textContent` (safe), reads `innerHTML` (escaped). Applied to all usernames, messages, and filenames.

Level 3

onDisconnect Auto-Cleanup

Firebase `onDisconnect().remove()` on `/status/{uid}`. If tab crashes, network drops, or user navigates away — status is wiped server-side automatically.

Level 3

Username-Based Auth

Login with username, not email. `/unames/{name}` → UID lookup. Email abstracted from login flow. Last username saved to localStorage.

Level 3

Zero Third-Party Scripts

No analytics, no pixels, no cookies, no fingerprinting. Only network requests: Firebase auth, database, and the 3 CDN resources (font, icons, nothing else).
